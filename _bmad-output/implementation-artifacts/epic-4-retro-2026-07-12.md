## The One Real Lesson — "end-to-end" was asserted from an incomplete model

Story 4.1 was titled *"add the nullable `game_changer` field **end-to-end**"* and its Dev Notes
explicitly stated *"Construction sites — no collateral changes needed."* Both were **wrong**.

The field flows through **four** write-path touch-points, not three:

1. `CardModel` (ORM) ✅ 4.1
2. `Card` schema (Pydantic) ✅ 4.1
3. `transform_scryfall_card` (extraction) ✅ 4.1
4. **`importer.py::_insert_batch` — the hand-listed INSERT `card_dict` + `on_conflict_do_update`
   `set_` mapping ❌ missed by 4.1**

Because #4 was missed, `game_changer` was extracted by the transformer and then **silently dropped
at the actual DB write**. The backfill re-import that 4.2 *documented* literally could not have
populated the column. Story 4.2's code review caught it; Brad expanded 4.2's scope to add the two
lines (mirroring the `power`/`toughness` write-path pattern that #4 already had).

**Why it was missable:** #4 is *mechanical duplication* — a hand-maintained column list — not a
type-driven site the way #1–#3 are. `mypy`/`ruff` can't flag an omission from a dict literal. The
precedent 4.1 was copying (`power`/`toughness`) actually touched all four, but the 4.1 story spec
only enumerated three and asserted completeness.

**A second instance of the same class surfaced during this retro:** committing the 4.2 `importer.py`
fix left the marketplace mirror `plugin/server/src/data/importers/importer.py` out of sync — the
`build-plugin-sync` pre-commit hook that the story notes assumed would auto-mirror **is not
installed as a git hook in this checkout** (`.git/hooks/pre-commit` does not exist). The mirror was
regenerated (`uv run python -m scripts.build_plugin`) and folded into the commit. Same failure mode:
a write-path/output touch-point silently skipped because an assumed automation didn't run.

## Action Items

1. **"End-to-end field" authoring checklist must enumerate all four write-path touch-points.**
   When a story claims a field flows end-to-end, the checklist grep must cover every INSERT/upsert
   **column list** (`importer.py::_insert_batch` — both the `card_dict` and the
   `on_conflict_do_update` `set_` mapping), not just the model/schema/transformer.
   *Owner: Brad (story authoring) · applies from Epic 5 onward.*

2. **Do not assume `build-plugin-sync` auto-mirrors — verify or run it explicitly.**
   `.git/hooks/pre-commit` is absent in this checkout, so any commit touching `src/` must be followed
   by `uv run python -m scripts.build_plugin` + a `diff -q src/... plugin/server/src/...` check before
   the commit is considered clean. (Consider `pre-commit install` to restore the hook.)
   *Owner: Brad · immediate.*

3. **Add a persisted-round-trip test for nullable bool fields (deferred debt, now recurring).**
   Both 4.1 review and the `power`/`toughness` precedent lack a real-SQLite test proving
   `game_changer = False` survives a DB write/read (only in-memory `CardModel` is asserted). The
   `importer.py` gap would have been caught by such a test. Low cost, high value now that a second
   nullable-bool field exists.
   *Owner: Brad · nice-to-have before Epic 5's scorer reads the field.*

### Backfill spot-check (verify-by-shape, not hardcoded names — GC list changes over time, NFR5)

```sql
SELECT count(*) FROM cards WHERE game_changer = 1;   -- expect ~50 (Feb 2026 is:gamechanger ≈ 53)
SELECT name  FROM cards WHERE game_changer = 1 ORDER BY name;   -- eyeball known staples
SELECT game_changer FROM cards WHERE name = 'Grizzly Bears';    -- expect 0/False
```

**Actual result (2026-07-12, `oracle_cards` re-import over the live 38,232-row DB):**

| State | Count |
|---|---|
| `TRUE`  | **53** ✅ (exact match to the Feb 2026 `is:gamechanger` list) |
| `FALSE` | 38,180 |
| `NULL`  | **0** (full corpus backfilled — no residual "unknown") |

Known Game Changers all read `True` (Cyclonic Rift, Rhystic Study, The One Ring, Mana Vault,
Vampiric Tutor, Force of Will, Smothering Tithe, Gaea's Cradle, …). `Grizzly Bears` (vanilla common)
reads `False`. The three-state contract survived the round-trip through the fixed `importer.py`
write path — direct confirmation that the Story 4.2 fix works, not just that it compiles.
