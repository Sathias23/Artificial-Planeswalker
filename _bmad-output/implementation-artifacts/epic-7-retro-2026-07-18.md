## Challenges & Lessons

### 1. The calibration work was orphaned by construction

The epic definition was frozen 2026-07-11; the G-R2 gate produced its five **named calibration
inputs** on 2026-07-17 — after the story list existed. Every story correctly scoped them out
("later Epic-7 calibration work"), and the epic ended with card_advantage saturation (80),
interaction railing (0/100), the Brawl mana_efficiency floor, almost_included dominance, and
**format-blind almost_included inflation** (Standard decks scoring `combo_potential=100` off
variants whose missing piece isn't Standard-legal) owned by nobody.

**Lesson → action item 1:** when a gate produces "named inputs," each input needs a home — a
story key, spec, or gate — *in the same commit that closes the gate*. A label with no key is
how work evaporates. Resolved this retro: the inputs now live in the keyed
`post-epic-7-calibration-gate` (sprint-status).

### 2. New I/O landed outside the error-contract guard

7.2's review found the epic's one substantive defect: `_provision_combos` issued three awaited
DB reads *after* the `DatabaseError → status="error"` block guarding the deck load. The 4-step
error pattern was faithfully copied — then new reads landed outside it. Blind + Edge hunters
converged independently; patched same day with a regression test.

**Lesson → action item 2:** same failure class as 6.1's construction-site lesson, different
surface — we enumerate result constructors, but nobody enumerated which guard covers each I/O
call. New discipline: when a story adds awaited I/O to an existing tool path, enumerate every
call against the error-contract guard that covers it.

### 3. Honest dead signals point at real data-layer debt

7.1 wired `unresolved_count` honestly knowing it is structurally zero — `DeckCard.card` is
required, so an orphaned `deck_cards` row (FK enforcement is off) crashes as a
`ValidationError` inside the repo, shared by every `get_deck_with_cards` caller. Documented
in three stories; the deferred data-layer orphan story had no tracking key — the same
evaporation risk as the calibration inputs. **→ action item 3** (keyed this retro).
