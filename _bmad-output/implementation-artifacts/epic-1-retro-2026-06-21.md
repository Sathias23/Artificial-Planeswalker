## 2. What Went Well (Strengths)

- **"Decisions Locked with Brad" up front (1.3 → 1.6).** Each story front-loaded its design forks (D-1.3a/b, D-1.4a–e, D-1.5a–g, D-1.6a–g) and resolved them with the project lead *before* dev, so implementation never re-litigated mid-build. This was the single biggest velocity driver.
- **Pattern reuse / compounding velocity.** Story 1.3 established the reusable skeleton — `build_server(session_factory)`, closure-registered async tools, the `seeded_card_db` fixture, the `create_connected_server_and_client_session` harness. Stories 1.4–1.6 reused it verbatim. 1.4's record: *"TDD red→green ran clean… no logic iteration was needed."*
- **Scope discipline.** Every story carries a "What this story IS — and is NOT" section and an anti-patterns list, and held to them. Data-layer changes stayed strictly additive across the whole epic (only adds: `bug_reports` slice, `CardSummary`, deck projections, `CardRepository.get_by_id`, `validate_deck`).

- **Run-and-capture verification + thorough Dev Notes.** Every story captured real command output and carried "Previous Story Intelligence" forward, so each story inherited the prior one's lessons.
- **Forward-built reuse.** `ConnectionFactory` (1.2) and `CardSummary` (1.4) were deliberately built ahead for Epic 2 — Epic 2's dependencies are already in place.

## 3. Challenges & Growth Areas (recurring patterns)

- **The same flaky test, all epic long.** `tests/integration/data/test_deck_repository.py::test_list_decks` ties on `created_at.desc()` with no secondary sort key. Found in 1.2, deferred, then flagged defensively in *every* subsequent story ("if it flakes, it's not your regression"). The fix is one line (`.order_by(created_at.desc(), id)`) and was never made.
- **Input-normalization / graceful-error lessons learned story-by-story** instead of once. 1.3 (empty `card_name` → `ilike('%%')` matched everything), 1.4 (`rarity=[]`, `format=""` bypassed guards), 1.5 (blank `deck_id`/`format` un-normalized; `DatabaseError` uncaught in 5 of 6 helpers). The same bug class recurred until 1.6 finally front-loaded it as locked decision D-1.6f.
- **Card-schema nullability — deferred three times.** 1.3, 1.4, and 1.6 reviews each flagged that `oracle_text`, `mana_cost`, `colors`, `legalities`, `games` are typed non-nullable while SQLite stores NULL for tokens/split cards/lands; `model_validate` would raise `ValidationError`. Each time punted to "a broader Card schema nullability audit" that has not happened.
- **FK enforcement OFF.** `add_card_to_deck` can insert orphan `deck_cards` rows (no `PRAGMA foreign_keys=ON` on the async engine); defended only by application-layer pre-validation in the tool. Structurally unguaranteed.
- **Baseline wasn't green at epic start.** Story 1.1's real time sink was not the restructure volume (67 renames, ~270 import rewrites) but two *pre-existing* defects: the unanchored `.gitignore` rule `data/` (which also matched `src/data/` and had silently kept `pagination.py` out of the repo) and the required-but-nullable `CardModel.printed_name` (broke 109 tests). The "Task 0" fixes (approved twice) were debt paid before AC4 could even be evaluated.

### Prep-gate completion note (2026-06-21)

All three action items and both optional wins landed, plus one extra fix surfaced during the pass. TDD throughout; suite **417 → 441** passing, `mypy --strict` clean (40 files), ruff clean on all touched files.

- **#1 (gate):** `None`-coercion validators on `Card` + `CardSummary` (`oracle_text`/`mana_cost`→`""`, `colors`/`games`→`[]`, `legalities`→`{}`), covering the `model_validate` ORM read path Epic 2 uses. **Empirical finding:** the current `data/cards.db` (38,232 cards) has **zero** NULLs in all five columns — the importer already coerces on write — so this is *defensive hardening* for a future/broader re-import, not a fix for a currently-failing read.
- **#2 (flaky test):** added the `id` tie-breaker to `DeckRepository.list_decks` and made `test_list_decks` deterministic by pinning explicit `created_at`s; added a dedicated tie-break determinism test. The flake reproduced live during the pass, then passed 5/5 repeated runs after the fix.
- **#3 (smoke):** agent ran a live MCP smoke against the real server/DB exercising all 10 tools (create→add→load→3 analysis→validate→remove→delete lifecycle) plus guard paths: `exists`, `card_not_found`/orphan-guard (no orphan row written), `invalid` (neither/both selector, `quantity 0`), `deck_not_found`, `not_in_deck`, idempotent re-delete, and `validate_deck` bad-`games`. All correct. The `ambiguous` name path could not be triggered live (single-word probes like "Goblin"/"Lightning" all hit real exact-named cards in the 38k corpus) — it remains covered by the in-memory harness. DB left pristine (smoke deck + smoke bug-report removed).
- **Extra (out of named scope):** fixed a latent null-passthrough in `transform_scryfall_card` — non-nullable fields used `.get(field, default)`, which only falls back on *missing* keys, so an explicit JSON `null` slipped through as `None`. Switched the non-nullable defaulted fields (`mana_cost`, `oracle_text`, `colors`, `color_identity`, `legalities`, `rarity`, `set`, `set_name`, `collector_number`, `cmc`) to `get(...) or default`; nullable `keywords`/`color_indicator`/etc. intentionally still preserve `None`. Stops NULLs at the source (belt-and-suspenders with the #1 schema gate).

### 🚨 Significant Discovery — affects Epic 2 readiness
The **card-schema nullability gap** (deferred 3× in Epic 1) escalates from a latent edge case to a **probable blocker** in Epic 2: the embedding builder reads `name + type_line + mana_cost + oracle_text + keywords` across the full ~60k-card corpus, where NULL `oracle_text` (split cards) and NULL `mana_cost` (lands/tokens) genuinely occur. **Resolved via prep-gate action item #1** — fix before Story 2.1.

## 7. Key Takeaways

1. **"Decisions Locked up front" is the velocity engine** — keep front-loading design forks with the project lead before dev (D-1.6f even pre-empted a recurring bug class this way).

3. **Recurring deferrals are a signal to just fix it** — the nullability gap and the flaky test each recurred across multiple stories before being addressed; treat the 2nd defer of the same item as a trigger to schedule the fix.
