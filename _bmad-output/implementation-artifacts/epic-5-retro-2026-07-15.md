## Challenges & Lessons

### 1. The 5.9 environment saga — trusted claims vs. verified state

The story's baseline note said the live DB was "imported AND backfilled." It wasn't: the
2026-07-10 games-union re-import predated the `game_changer` migration and the column was missing
entirely. The repair then uncovered two deeper importer defects:

- **Stale-printing accumulation:** Scryfall's preferred printing per oracle identity churns between
  bulk snapshots; each refresh inserts rows under new printing ids while old rows persist
  (~13k stale rows; 4,711 names — including Rhystic Study — resolved first-by-id to a stale row).
- **Silent import errors:** 36 cards (Blood Crypt, Hallowed Fountain, …) errored with no per-card
  diagnostics and kept stale/NULL data.

Both were hand-repaired operationally and logged as importer-owned deferred items (out of 5.9's
frozen scope). Epic 4's retro had recorded 0 NULLs on 2026-07-12; the 2026-07-14 refresh
reintroduced 12,992 — i.e. **the rebuild path itself re-damages the DB on every run.**

**Sathias's ruling:** the DB is *designed* to be rebuilt every season — the concern is not DB state
but the rebuilder. Consequence (team consensus): rebuild-friendly design makes the import path
load-bearing infrastructure; fix it once, correctly, and "just rebuild it" becomes true by
construction. → Pre-Epic-6 Importer Gate (action items G-I1..G-I3).

**Root lesson (same class as Epic 4's "end-to-end" miss):** a written claim about external state
was trusted instead of checked. One `SELECT count(*) … WHERE game_changer IS NULL` at story start
would have caught it immediately. → Action item 4.

### 2. DFC / multi-face name handling recurred three times

5.3 (multi-face cards store `oracle_text=""`; null-face crash patched), 5.6 (commander gate skipped
front-face normalization — review patch), 5.9 (two committed decklists carried front-face-only
names failing exact lookup). Same hazard, three stories. `_name_keys` (5.6) is the reference
normalization; 6.2's Spellbook-name matching must reuse it.

### 3. Claims outrunning verification (documentation accuracy)

The stale 5.9 baseline note; 5.5's story self-contradicting on hook status; 5.3 and 5.5 checking
off "asserts have failure messages" while every assert was bare; docstring-overclaim findings
dominating late-epic reviews (5.7's entire review was two docstring patches). Standing agreement:
**drop the claim or write the message** — extended from asserts to story notes and baseline
assumptions.

### Calibration honesty — what the green benchmark does and doesn't prove

Raised by Sathias: "hard to judge until we test it against real use cases." The team agrees the
gate proves scoring *shape*, not real-world validity:

- 5.1's decklists were **reconstructed from memory, not sourced verbatim** (documented Decision).
- The cEDH combo variants were **hand-built** — the real Spellbook snapshot arrives in Epic 6.
- Ten decks total. Ordering and diff-sensitivity are proven; generalization is not.

Real proof-out is scheduled at Epic 7 (tool on real decks) — but `score()` is pure, so an informal
real-deck sanity pass is cheaply available *now* (action item 5). Notable calibration events for
the record: Commander weights re-spread (combo_potential .15→.30, speed .10→.15) after the Talrand
precon outscored the Tymna cEDH list under v3 weights; `CEDH_TUTOR_MIN` 4→3 (FR6's tutor definition
excludes battlefield/library-exile tutors, so real cEDH lists undercount); profiles bumped to
commander-v4 / standard-v4.
