## Intent

**Problem:** `pyproject.toml` still carries the pre-MCP-pivot package shape: orphaned runtime
deps from the deleted legacy/Postgres stack (`anthropic`, `openai`, `asyncpg`), an unused
`tenacity`/`python-dotenv` pair, a now-dead `[dependency-groups] legacy` block (the `legacy/`
tree was hard-deleted in the prune), a "built with PydanticAI" description, a placeholder author
email, and no console entry point. For a public release this misrepresents the project and bloats
a default `uv sync`.

**Approach:** Shrink the runtime dependency set to what `src/` actually imports, move `logfire`
into an optional `observability` extra, delete the dead legacy group, and correct package metadata
(description, author, console script). Regenerate the lockfile and prove suite + lint + type-check
still pass. No runtime behaviour changes.

## Spec Change Log

## Design Notes

- `[project.optional-dependencies]` (PEP 621 extra) chosen over a `[dependency-groups]` entry for
  observability: logfire is a runtime-optional integration a *consumer* may want
  (`pip install …[observability]`), whereas the deleted `legacy` group was dev/reference-only.
- `tenacity`/`python-dotenv` stay reachable transitively (logfire → tenacity; other deps pull
  dotenv) — dropping the *direct* declarations is correct because no first-party code imports them.
- Entry point must reference `src.*` (only `src/` is in the wheel); `scripts/` helpers cannot be
  console scripts.
