## Intent

**Problem:** Artificial Planeswalker installs only via the Claude Code plugin or manual per-client MCP config; OpenAI Codex CLI users (plugin system since v0.117.0, ~2026-03) have no supported path to the MCP server or the four MTG skills.

**Approach:** Extend the committed `plugin/` tree with Codex plugin artifacts generated by `scripts/build_plugin.py` (`.codex-plugin/plugin.json` + `codex-mcp.json`), hand-commit a repo-scoped Codex marketplace at `.agents/plugins/marketplace.json`, and add an "OpenAI Codex" block to the README's *Connect your client* section ‚Äî including a manual `config.toml` fallback that works independently of the plugin path. Gitignore the BMAD `.agents/skills/` dump so the tree stays clean.

## Boundaries & Constraints

**Always:**
- Derive the Codex manifest from `pyproject.toml [project]` exactly like `_plugin_json` ‚Äî no second metadata source.
- All generated JSON goes through `_write_json` (LF-forced) so Windows and CI builds stay byte-identical; add new generated paths to the managed-clean list in `build()`.
- `plugin/skills/` and `plugin/server/` are shared by both clients ‚Äî copy skills unchanged; existing Claude artifacts (`.claude-plugin/plugin.json`, `.mcp.json`) must be byte-identical after this change.
- Codex MCP config uses the documented snake_case `mcp_servers` wrapper and anchors the launch with `cwd: "./server"` + `uv run python -m src.mcp_server` (Codex does NOT substitute `${CLAUDE_PLUGIN_ROOT}` ‚Äî openai/codex#19372).
- README labels the Codex plugin path **experimental / not yet live-verified**; the manual `~/.codex/config.toml` snippet (absolute `--directory` path, mirroring the Cursor block) is the verified path.
- Keep `.agents/plugins/` trackable ‚Äî ignore only `.agents/skills/`.

**Ask First:**
- Any edit to the four SKILL.md files (bodies or frontmatter).
- Any change to the existing Claude `.mcp.json` generator.
- Removing the "experimental" README label ‚Äî only after Brad's live Codex smoke test.

**Never:**
- Don't rename `mcp__artificial-planeswalker__*` tool references inside skills.
- Don't blanket-ignore `.agents/` (would swallow the marketplace file).
- Don't add Codex-specific code to `src/` ‚Äî the server is already client-agnostic.
- Don't invent undocumented manifest fields; stick to the researched schema.

## Design Notes

- **Why `cwd` instead of `--directory`:** Codex performs no `${...}` substitution in plugin MCP configs (openai/codex#19372); plugin paths are documented as `./`-relative to plugin root, so `cwd: "./server"` + plain `uv run` is the least-speculative anchor. A `$PLUGIN_ROOT` wrapper script was rejected (env propagation undocumented; `sh` breaks native-Windows Codex).
- **Unverified externals** (‚Üí Brad's live smoke, hence the experimental label): `cwd` support inside plugin MCP config; marketplace `local` path resolving from repo root.
