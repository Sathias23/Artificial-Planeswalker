Distilled directly, no elicitation. Slug=deck-power-assessment (the thing specced, = spine name).
- (note) Why = opportunity+vision: a new MCP capability that scores a saved deck and lets its owner compare two decks / two versions and see what changed and whether it got stronger. Deterministic, explainable, honest about uncertainty. Commander-first (WotC Brackets) + Standard. Headline: relative comparison of the owner's own decks; comparison is caller-side (run twice, diff).
- (capability) CAP-1 Assess a saved deck. intent: caller runs assess_deck_power(deck_id, format?) to get a power read — Commander Bracket 1-5 + for-format 0-100 + descriptive tier label + 7-dimension vector (speed/consistency/resilience/interaction/mana_efficiency/card_advantage/combo_potential). success: a Commander deck returns a bracket + all 7 dims + label; a Standard deck returns a heuristic for-format score + all 7 dims + label (no bracket).
- (capability) CAP-2 Resolve format gracefully. intent: infer format from the deck when omitted; reject unsupported formats without crashing. success: omitted format is inferred (commander|standard); an unsupported/unrecognized format returns a graceful error result naming the supported formats, never an exception.
- (capability) CAP-3 Deterministic, diffable output. intent: identical deck+snapshot+cached-combo-data always produce identical structured output so a caller can diff two runs; a meaningful edit produces a correctly-directioned dimension delta. success: regression test shows byte-identical JSON for identical inputs; adding a Game Changer or combo piece moves the expected dimension in the expected direction.
- (capability) CAP-4 Detect combos via Commander Spellbook. intent: find real two-card-infinite/loop combos in the deck and feed them into both the Commander Bracket floor and the combo_potential dimension. success: a deck containing a known included combo surfaces it in flags.combos and it sets/raises the Bracket floor per the WotC two-card-infinite trigger.
- (capability) CAP-5 Detect Game Changers. intent: determine per-card Game Changer membership and the deck count, feeding the Commander Bracket floor per the WotC decision tree. success: a deck with N Game Changers lists them in flags.game_changers and floors the Bracket (1-3 GC -> B3, etc.); NULL GC data does not silently floor to B2.
- (capability) CAP-6 Stay honest under degradation. intent: every score carries a categorical confidence (low|medium|high) + named reasons; unresolved cards, unreachable Spellbook, or unpopulated GC data lower confidence rather than crashing or silently scoring zero. success: Spellbook unreachable -> combo_data_unavailable/stale reason + degrade; unresolved cards -> cards_unresolved + count; NULL game_changer -> game_changer_data_unavailable; no crash, no silent zero, in every case.
- (capability) CAP-7 Dual output. intent: return both a human-readable formatted summary and the raw structured assessment (with a flags block surfacing the exact Game Changers / combos / structural gaps / triggers / cedh_candidate that drove the result). success: one AssessDeckPowerResult carries both a summary string and the structured assessment; flags name the specific cards/combos/gaps behind the score.
- (constraint) Determinism is the load-bearing constraint (NFR1): no randomness in v1; identical deck + Scryfall snapshot + cached combo data -> identical scores. This is what makes caller-side diffing trustworthy. Enforced at the output boundary by the spine's AD-8 (sorted lists, int scores, no call-time clock).
- (constraint) Commander-first; Standard is deliberately shallower — heuristic-only (curve/interaction/Karsten-60/combos), NO Bracket and NO meta-tier percentile. Standard's for-format score always ships with a descriptive tier label so no bare number is presented.
- (constraint) Single live external dependency: Commander Spellbook find-my-combos (keyless, MIT). Must be cached >=24h with polite throttling + 429 backoff and degrade gracefully (mirrors the index_unavailable pattern). It is the only non-local call in an otherwise offline server.
- (constraint) Game Changer data requires a data-layer change: a nullable game_changer field added to CardModel/Card + transform_scryfall_card, a hand-written migration (no Alembic), and a heavy Scryfall re-import to backfill. NULL = unknown and lowers confidence; readers never coalesce NULL to False. GC freshness couples to import cadence.
- (constraint) cEDH (Bracket 5) is flagged as CANDIDACY only, never auto-asserted from cards alone. Intent is unobservable — Brackets are officially 'not an exact science'; auto-classification carries an irreducible error band reflected in mandatory confidence.
- (constraint) Architecture conformance: the feature is bound by the adopted architecture spine AD-1..AD-12 — async MCP tool (sibling to the analysis tools), pure deterministic scorer in src/logic vs impure I/O edge, framework-free src/data & src/logic, one canonical ComboRecord + cache key, closed-enum confidence vocabulary. Downstream stories cite the AD IDs, which are stable.
- (constraint) Power is framed as an ESTIMATE, not a verdict that a deck is 'better' — non-cEDH Commander is social-first. Output never presents a bare number and never claims a deck is better; a higher score is a power estimate only.
- (constraint) v1 dimension signal->0-100 mappings + aggregate weights are hand-tuned, documented, adjustable, and validated against a committed calibration benchmark; composing that benchmark set (WotC precons ~Bracket 2, known cEDH lists flagged as candidates) is the FIRST implementation task and is the scorer's acceptance signal.
- (decision) Non-goals (PRD 2.1/8 + addendum A): Monte Carlo goldfish simulation; ML/embedding scoring; Limited/Draft assessment; calibrated cross-format ABSOLUTE score + per-format offset; 60-card meta-tier percentile (MTGTop8/Goldfish scraping); EDHREC enrichment; a dedicated compare_decks tool (comparison is caller-side).
- (note) Success signal: Brad assesses a Commander deck (e.g. Bracket 3, for-format 68, dimension breakdown, 2 Game Changers + 1 late combo noted), swaps three cards, re-assesses the new deck_id, and diffs the two structured results to read combo_potential up / speed up / mana_efficiency down — the exact 'did my edit make it stronger and what changed' question, answered. Acceptance: WotC precons land ~Bracket 2 and known cEDH lists flag as candidates in the committed benchmark; identical inputs -> identical output.
